# Teste para Vaga de Desenvolvedor Full Stack

## **Premissas**

- O teste é dividido em duas etapas: **Front-end e** **Back-end**.
- O prazo máximo para entrega é de **7 dias corridos**, contados a partir da data de envio deste e-mail.
- A avaliação será feita de acordo com sua **experiência atual** e seu **raciocínio lógico**.
- Sempre que possível, recomendamos desenvolver seus **próprios componentes**, em vez de utilizar bibliotecas prontas. O uso de bibliotecas externas não é proibido nem invalida o teste.
- Em caso de dúvidas, **tome a decisão que considerar mais adequada**. Registre as dúvidas e as escolhas feitas para entregar junto ao teste final.
- **O Front-end é o ponto de maior ênfase na avaliação.** Espera-se que ele esteja completo, funcional e em conformidade com o design proposto, sendo considerado determinante para a análise do teste.

---

## **Front-end**

- **Obrigatório** utilizar **React com TypeScript** e **styled-components** para o desenvolvimento.
- **Obrigatório** utilizar **Contexts** e **Hooks** na implementação.
- A interface deve seguir o design proposto: [Figma](https://www.figma.com/design/Fx54OPLCz0r0X2jNWrEPSl/Teste-DEV-Goalfy?node-id=0-1&t=DHTQaQu313mTge1R-1).
- Implementar um **CRUD de clientes**, incluindo:
    - Modal para cadastro de novos clientes;
    - Listagem dos clientes cadastrados.
- Validar os campos no cadastro de clientes (ex.: campos obrigatórios, e-mail e CEP).
- Utilizar uma **API pública de CEP** à sua escolha para preencher automaticamente os campos de endereço.

---

## **Back-end**

- **Obrigatório** utilizar **Java** ou **Node.js**.
- Criar um serviço **REST** para integração com o Front-end, implementando as operações de **CRUD (GET, POST, PUT, DELETE)**.
- Persistir os registros em um **banco de dados** de sua escolha.
- Estruturar o projeto com base nos princípios de **DDD (Domain-Driven Design)**.

---

## **Entrega**

- O projeto deve ser disponibilizado em um **repositório público no GitHub**, permitindo sua visualização e execução.
- O repositório deve conter um **README detalhado**, incluindo:
    - Descrição do projeto;
    - Tecnologias utilizadas;
    - Instruções para instalação e execução do projeto (tanto Front-end quanto Back-end);
    - Passos para configurar dependências, variáveis de ambiente e banco de dados (quando necessário);
    - Exemplos de requisições para a API (se aplicável).
